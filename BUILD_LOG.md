# OpenClaw 智能选股 & 模拟交易系统

> 基于 OpenClaw AI Agent + GPT 5.4 大模型，构建的 A 股/港股智能选股与模拟交易系统。
> 通过 Telegram 进行自然语言交互，支持多策略选股、模拟买卖、持仓管理和定时推送。

## 项目背景

传统量化选股系统门槛高、交互复杂，普通用户难以上手。本项目利用 OpenClaw AI Agent 框架的自然语言理解能力，将专业的技术分析策略封装为简单的聊天指令，用户在 Telegram 上发一句"选股"即可获得专业级的选股建议，发"买入 600519 100股"即可完成模拟交易。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| AI 框架 | OpenClaw 2026.3.13 | Node.js/TypeScript，AI Agent 运行时 |
| 大模型 | GPT 5.4 | 自然语言理解、指令解析、智能决策 |
| 数据源 | AKShare（Python） | 新浪财经接口，A 股/港股实时行情与历史 K 线 |
| 股票池 | 沪深300成分股 | 中证指数公司官方成分股，默认扫描前 100 只 |
| 技术分析 | ta 库 | MACD、KDJ、RSI、SMA 等 50+ 技术指标 |
| 数据处理 | pandas | 时间序列数据清洗与计算 |
| 消息通道 | Telegram | 实时消息推送与指令交互 |
| 持久化 | JSON 文件 | 轻量级账户数据存储 |
| 定时调度 | OpenClaw Heartbeat | 每 30 分钟心跳检查，驱动定时任务 |

## 核心功能

### 1. 智能选股（screener.py）

从沪深300成分股中，基于技术指标自动筛选出近期出现买入信号的股票。

| 策略 | 命令 | 选股逻辑 | 适用场景 |
|------|------|---------|---------|
| MACD 金叉 | `--strategy macd` | 近 3 天内 DIF 上穿 DEA | 趋势反转初期 |
| 均线多头 | `--strategy ma` | 收盘价 > MA5 > MA10 | 上升趋势确认 |
| KDJ 金叉 | `--strategy kdj` | 近 3 天内 K 上穿 D，J < 90 | 超卖反弹 |
| 综合策略 | `--strategy combined` | MACD 金叉 + 均线多头同时满足 | 高确定性信号 |

数据流程：拉取成分股列表 → 逐只获取 90 天日 K 线 → 计算技术指标 → 策略条件判断 → 输出符合条件的股票及关键指标值。

### 2. 模拟交易（trade.py）

| 功能 | 说明 |
|------|------|
| 初始资金 | 5 万虚拟资金 |
| 买入 | 支持市价/限价，自动计算手续费（万三） |
| 卖出 | 自动计算手续费（万三）+ 印花税（千一），输出盈亏 |
| 持仓查询 | 实时拉取当前股价，计算浮动盈亏和收益率 |
| 账户总览 | 可用资金 + 持仓市值 = 总资产，对比初始资金算总收益 |
| 交易记录 | 完整的买卖流水，支持按股票代码筛选 |
| 成本计算 | 加权平均法，与真实券商算法一致 |

### 3. Telegram 自然语言交互

用户无需记忆任何命令，直接用中文对话：

| 用户消息 | 系统动作 |
|---------|---------|
| 选股 / 扫描 / scan | 运行综合策略扫描 |
| 选股 macd / kdj / 均线 | 运行对应策略 |
| 港股选股 | 切换港股市场扫描 |
| 买入 600519 100股 | 以实时价格模拟买入 |
| 卖出 600519 50股 | 模拟卖出并计算盈亏 |
| 持仓 / 仓位 | 查看当前持仓及浮动盈亏 |
| 账户 / 资金 | 查看账户总资产 |
| 交易记录 | 查看历史买卖流水 |

### 4. 定时任务（外部调度）

| 任务 | 触发时间 | 内容 |
|------|---------|------|
| 收盘扫描 | 每个交易日 15:30 | 由外部调度调用 `auto_analyzer.py` |
| 盘中监控 | 交易日 9:30-15:00 每小时 | 由外部调度调用 `auto_analyzer.py` |
| 周末复盘 | 周六 10:00 | 当前仓库未实现，需单独补任务 |

## 开发过程中的技术决策

| 问题 | 决策 | 原因 |
|------|------|------|
| 东方财富 API 全市场请求断连 | 改用沪深300成分股作为候选池 | 300 只大盘蓝筹覆盖主流标的，请求量可控 |
| 东方财富 K 线接口并发限流 | 切换新浪财经接口 | 新浪接口无频率限制，数据质量一致 |
| 新浪接口底层 V8 引擎不支持多线程 | 改为顺序执行 | mini_racer 在 ThreadPoolExecutor 下 crash，顺序执行稳定可靠 |
| 30 天数据 MACD 指标未收敛 | 数据窗口扩大到 90 天 | MACD 慢线需要 26 周期 + 信号线 9 周期，至少 35 个交易日 |
| 策略仅看当天信号命中率低 | 放宽为近 3 天窗口 | 避免因扫描时间差错过有效信号 |

## 后续迭代计划

### Phase 2：市场扩展

- 港股数据源优化（当前为 fallback 硬编码列表，计划接入港股通实时数据）
- 美股支持（接入 Yahoo Finance / Alpha Vantage API）

### Phase 3：策略增强

- 布林带突破策略、量价配合策略、RSI 超卖反弹策略
- 历史回测功能（基于过去 1 年数据验证策略胜率）
- AI 辅助选股（结合 GPT 5.4 分析财报、新闻舆情）

### Phase 4：模拟盘 → 实盘

- 对接券商量化交易接口（QMT / 恒生 PTrade）
- 新增 `broker.py` 统一券商适配层，支持实盘下单
- 通过环境变量 `REAL_MODE=true` 一键切换模拟/实盘
- 风控模块：单笔最大仓位限制、日亏损熔断、止盈止损自动执行
- 港美股实盘对接 Interactive Brokers（盈透证券）Python SDK

### Phase 5：飞书集成

- 飞书审批流：实盘交易需飞书审批确认后执行
